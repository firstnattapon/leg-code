3Diagnosis — รากของปัญหา

RC1 — Webull API ไม่มี DNA และ custom ledger w=0.6

  1. API ให้ข้อมูล broker
  2. DNA และเหตุผลถูกเก็บใน Firestore

RC2 — CSV มี unnamed index และ nullable rows w=0.4

  1. export รวม index
  2. ERROR/non-price rows ไม่มีค่าครบ

RC3 — Streamlit rerun ไม่มี dependency graph w=1.0

  1. widget เปลี่ยนแล้วสคริปต์ rerun
  2. ผล downstream อาจค้างถ้าไม่ hash และ invalidate

RC4 — CSV ใช้ทุก quote แต่เงินจริงต้องอาศัย fill w=1.0

  1. ORDER_SUBMITTED ไม่ยืนยัน cash
  2. ต้องมี execution price และ position reconciliation

RC5 — Cloud ถือ service account และ UAT capability w=1.0

  1. repository เป็น public
  2. app ต้อง private และ output ต้อง redact

RC6 — tests เดิมยังไม่รู้จัก LEGO workflow w=1.0

  1. ไม่มี registry/state machine
  2. ไม่มี gating และ audit tests

สมการวัดผล

Stage completionsuccessful_stage_runs / 17 → target 1.0
Contract conformancepassed_contract_assertions / all_contract_assertions → target 1.0
Regression pass ratepassed_tests / total_tests → target 1.0
Goal attainmentA = G_achieved / G_target
Progress ratioP = (G_n − G_0) / (G_target − G_0)
Weighted qualityS = 0.40·stage + 0.35·contract + 0.25·tests

🔁 Control loop

Errore_n = G_target − G_n
Convergencer_n = |e_n| / |e_(n−1)|

หยุด: เมื่อ |e_n| ≤ ε = 0.01 · K = 0.5 · n_max = 5

สถานการณ์วนกลับไปการแก้
r_n < 1Step 5แก้ acceptance failure รอบถัดไป
r_n ≈ 1Step 3ทบทวน root cause และ contract
r_n > 1Step 2ทบทวน problem/data boundary
ครบหลายรอบยังห่างStep 1ปรับเป้าหรือแบ่ง release
